Overview

Every PZ mod lives in its own folder. The game detects a mod by finding a mod.info file. Build 42 introduced versioned folders — your content goes inside a 42/ subdirectory so the game knows which build it targets.

Version Folders

  • 42/ — Build 42 content. The game loads the highest compatible version folder.
  • common/ — Resources shared across all game versions. Good for large assets (textures, models, sounds). Can be empty.

media/ Subdirectories

DirectoryContents
scripts/Item definitions, recipes, vehicle scripts, fixing blocks.
lua/client/Client-side: UI, context menus, timed actions.
lua/server/Server-side: spawning, farming, weather events.
lua/shared/Shared logic (loaded first). Translations live here.
textures/PNG images. Item icons need the Item_ prefix.
models/3D model files for weapons, clothing, vehicles.
sound/Audio in .wav or .ogg format.

tiledefNoTiledef name and ID.

Translation Files

Live under media/lua/shared/Translate/EN/. Format:

Items_EN = {
    ItemName_MyItem = "My Custom Item",
}

Each language gets its own subfolder. The game falls back to EN if a translation is missing.

Item Icon Naming

Texture files for item icons must start with Item_ — for example, Item_MyItem.png. Place them in the textures/ folder.

Testing Your Mod

Place your mod folder in %UserProfile%\Zomboid\mods\ or subscribe via the Steam Workshop. The game scans for mod.info files on startup.
